# SellerInsight Frontend

SellerInsight Frontend는 스마트스토어 판매자가 주문 CSV와 집계 데이터를 바탕으로 매출 흐름, 주문 추이, 품절 위험, 장기 미판매 상품, 추천 액션을 한 화면에서 확인할 수 있도록 만든 커머스 운영 콘솔입니다.

이 저장소는 `SellerInsight Backend`가 제공하는 API와 데이터 처리 결과를 사용자 화면으로 연결하는 프론트엔드 애플리케이션입니다.

백엔드가 CSV 업로드, 데이터 정규화, 일별 지표 집계, 인사이트 생성을 담당한다면, 이 프론트엔드는 그 결과를 판매자와 운영자가 실제 업무 흐름에 맞게 탐색할 수 있는 UI로 제공합니다.

## 화면 미리보기

### 메인 화면

![메인 화면](assets/readme/메인%20페이지.png)

첫 화면에서는 API 서버와 데이터베이스 상태, 현재 선택된 판매자 컨텍스트, CSV 업로드와 대시보드로 이동하는 빠른 실행 버튼을 제공합니다. 하단에는 데이터가 수집되고 인사이트로 이어지는 흐름을 단계별로 보여줍니다.

### 파일 업로드 화면

![파일 업로드 화면](assets/readme/파일%20업로드%20페이지.png)

판매자는 주문 CSV 파일을 선택해 업로드할 수 있습니다. 업로드 이후에는 Import Job ID, 처리 상태, 전체 행 수, 성공/실패 행 수를 확인하며 적재 결과를 추적할 수 있습니다.

### 판매자 정산 대시보드

![판매자 정산 대시보드](assets/readme/판매자%20결과%20대시보드.png)

대시보드는 최신 집계일 기준의 핵심 지표를 요약합니다. 총 주문 수, 총 매출액, 품절 상품 수, 장기 미판매 상품 수를 카드 형태로 보여주고, 최근 7일 매출/주문 추이는 라인 차트로 확인할 수 있습니다. 우측에는 최근 생성된 인사이트 목록이 함께 표시됩니다.

### 판매자 인사이트 화면

![판매자 인사이트 화면](assets/readme/판매자%20인사이트%20페이지.png)

인사이트 화면에서는 날짜별로 생성된 문제 상황을 탐색할 수 있습니다. 좌측 목록에서 인사이트를 선택하면 우측 상세 영역에서 심각도, 요약 설명, 추천 액션, 판단 근거 데이터를 확인할 수 있습니다.

## 사용자 흐름

1. 우측 상단의 `API 인증 설정`에서 계정을 입력합니다.
2. 판매자 계정으로 로그인하면 연결된 판매자가 자동으로 선택됩니다.
3. 관리자 계정은 `Account Status`에서 확인할 판매자 ID를 직접 불러올 수 있습니다.
4. `Import CSV`에서 주문 CSV 파일을 업로드하고 처리 결과를 확인합니다.
5. 집계와 인사이트가 생성된 뒤 `Dashboard`에서 핵심 지표와 최근 인사이트를 확인합니다.
6. `Insights`에서 날짜별 인사이트를 선택해 추천 액션과 근거 데이터를 검토합니다.

## 주요 화면 구성

| 메뉴 | 사용자 관점의 역할 |
| --- | --- |
| `Overview` | 시스템 상태, 현재 판매자 선택 상태, 주요 작업 바로가기를 확인하는 시작 화면 |
| `Account Status` | 현재 로그인 계정과 연결된 판매자 컨텍스트를 확인하는 화면 |
| `Credentials` | 네이버 커머스 연동용 Client ID/Secret을 저장하고 토큰 발급을 테스트하는 화면 |
| `Import CSV` | 주문 CSV를 업로드하고 적재 작업 상태를 추적하는 화면 |
| `Dashboard` | 최신 판매 지표, 7일 추이, 최근 인사이트를 요약해서 보는 화면 |
| `Insights` | 특정 날짜의 인사이트 상세와 추천 액션을 확인하는 화면 |
| `Pipeline Runs` | 운영자가 샘플 데이터 생성, 일별 파이프라인 실행, 실행 이력 확인, Lock 해제를 수행하는 관리자 화면 |

## 계정 안내

로컬/데모 환경에서는 Basic Auth 기반 계정을 사용합니다.

| 역할 | 계정 | 용도 |
| --- | --- | --- |
| Admin | `admin / admin-1234` | 샘플 데이터 생성, 파이프라인 실행, 전체 판매자 운영 확인 |
| Seller | `seller-demo / seller-demo-1234` | 연결된 판매자의 CSV 업로드, 대시보드, 인사이트 확인 |

인증 정보와 선택한 판매자 정보는 브라우저 로컬 스토리지에 저장되어 새로고침 후에도 유지됩니다. 로그아웃하면 저장된 인증 정보와 판매자 컨텍스트가 초기화됩니다.

## 관리자 사용 시나리오

관리자는 `Pipeline Runs` 화면에서 데모 데이터를 만들고 일별 파이프라인을 실행할 수 있습니다.

- `Sample Data Bootstrap`에서 `default` 또는 `large` 시나리오를 선택해 샘플 데이터를 생성합니다.
- `Run Daily Pipeline`에서 날짜를 지정해 전체 판매자 대상 집계와 인사이트 생성을 실행합니다.
- `Execution History`에서 실행 상태, 처리 판매자 수, 실패 수, 소요 시간을 확인합니다.
- 실행 이력을 클릭하면 판매자별 처리 로그와 생성된 인사이트 수를 볼 수 있습니다.
- 문제가 생긴 날짜는 `Danger Zone`에서 파이프라인 Lock을 강제로 해제할 수 있습니다.

## 판매자 사용 시나리오

판매자는 업로드부터 인사이트 확인까지 다음 순서로 이용합니다.

- `API 인증 설정`에서 Seller 계정으로 로그인합니다.
- 상단 바에서 현재 연결된 판매자명과 외부 판매자 ID를 확인합니다.
- `Import CSV`에서 주문 CSV를 업로드합니다.
- `Dashboard`에서 최신 주문 수, 매출액, 품절 상품, 장기 미판매 상품을 확인합니다.
- `Insights`에서 날짜를 선택하고 인사이트별 추천 액션을 검토합니다.

## 로컬 실행

```bash
npm install
npm run dev
```

기본 접속 주소는 `http://localhost:5173`입니다.

로컬 개발 환경에서는 Vite 프록시가 `/api` 요청을 `http://localhost:8080` 백엔드로 전달합니다. 별도 백엔드 주소를 사용해야 한다면 `.env.example`을 참고해 프로젝트 루트에 `.env.local`을 만들고 `VITE_API_BASE_URL`을 설정합니다.

```env
VITE_API_BASE_URL=""
```

## 백엔드 연동 전 확인할 점

- 백엔드 서버가 `http://localhost:8080`에서 실행 중이어야 합니다.
- 대시보드의 추이 그래프는 CSV 업로드만으로 바로 표시되지 않을 수 있습니다. 대상 날짜의 일별 집계와 인사이트 생성이 완료되어야 합니다.
- 테스트용 CSV 파일이 필요하면 백엔드 README의 CSV 필수 헤더와 업로드 흐름을 기준으로 준비합니다.
- 관리자 화면의 샘플 데이터 생성은 기존 데이터를 초기화할 수 있으므로 데모/로컬 환경에서만 사용하는 것을 권장합니다.

## UI 특징

- 좌측 사이드바에서 판매자 작업공간과 관리자 콘솔을 분리했습니다.
- 상단 바에서 현재 인증 계정과 선택된 판매자 상태를 계속 확인할 수 있습니다.
- 성공, 실패, 처리 중, 심각도는 색상 배지로 구분해 빠르게 스캔할 수 있습니다.
- 대시보드는 숫자 카드와 차트를 함께 배치해 최신 상태와 기간 추이를 동시에 볼 수 있게 구성했습니다.
- 인사이트 상세 화면은 목록과 상세 패널을 나누어 문제 탐색, 추천 액션 확인, 근거 데이터 검토가 한 흐름으로 이어지도록 설계했습니다.
